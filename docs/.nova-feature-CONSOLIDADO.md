# ✨ Workflow: Nova Feature

## Quando Usar
Implementação de nova funcionalidade seguindo TDD.

## Passos

### 1. Branch
Ver `/git-flow` → Branch: `feat/nome-da-feature`

### 2. Planejamento
- [ ] Criar ADR se decisão arquitetural
- [ ] Check de privacidade (Local-First)
- [ ] Análise de impacto (breaking changes?)

### 3. TDD Red → Green → Refactor
- [ ] **Red**: Testes que definem comportamento (FALHAM)
- [ ] **Green**: Implementação mínima (PASSA)
- [ ] **Refactor**: Melhorar sem quebrar testes

### 4. Quality Gate
```bash
npm run check        # Lint + types
npm run test:quick   # Testes falhos
```

### 5. Commit & Merge
```
feat(escopo): adiciona funcionalidade X
```
Ver `/git-flow` para merge
